Homework 1: Tasks for mr414

Your personal task is to develop a system for cafes and their cakes.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Cake. With the following private fields:

  1. String name
  2. String description
  3. String ingredients
  4. int amount
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Cafe. With the following private fields:

  1. int id
  2. String name
  3. String address
  4. List<Cake> cakes
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Cafe> cafes in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one cafe with two cakes. [4 marks]

Create controller co2103.hw1.controller.CafeController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/cafes" should show view cafes/list. [2 marks]
    • Add the list of all cafes from co2103.hw1.Hw1Application to the model as attribute cafes. [2 marks]
    • Create the corresponding JSP to list the name and address of each cafe followed by a link to URL "/cakes?cafe={id of cafe}", e.g., "/cakes?cafe=3". [4 marks]
  2. Requests for URL "/newCafe" should show view cafes/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="cafe" and action="/addCafe" and inputs for all fields of class Cafe except for cakes. [5 marks]
  3. POST requests for URL "/addCafe" should add a new cafe to the list of all cafes in co2103.hw1.Hw1Application and redirect to /cafes. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.CakeController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/cakes" should show view cakes/list. [2 marks]
    • Based on request parameter cafe determine the corresponding cafe from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all cakes of that cafe to the model as attribute cakes. [2 marks]
    • Create the corresponding JSP for the view to list the name, description, ingredients, and amount of each cake. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newCake?cafe={id of cafe whose cakes are listed}", e.g., "/newCake?cafe=3". [4 marks]
  2. Requests for URL "/newCake" should show view cakes/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="cake" and action="/addCake" and inputs for all fields of class Cake. [6 marks]
    • Also add the request parameter cafe with the ID of the current cafe. [4 marks]
  3. POST requests for URL "/addCake" should add a new cake to the list of cakes of the cafe identified by request parameter cafe and then redirect to /cafes. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Cafe and make it available in controller CafeController. [4 marks]

When adding a new cafe validate that

  1. The value of field id is unique within the list Hw1Application.cafes. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field address is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Cake and make it available in controller CakeController. [4 marks]

When adding a new cake validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field description is not empty. [2 marks]
  3. The value of field ingredients is either "Sugar", "Butter", or "Flour". [3 marks]
  4. The value of field amount is between (including) 0 and 15. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
